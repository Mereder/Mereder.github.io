你有两个篮子，每个篮子可以携带任何数量的水果，但你希望每个篮子只携带一种类型的水果。

三角形最小路径和

给定一个三角形，找出自顶向下的最小路径和。每一步只能移动到下一行中相邻的结点上。

[

[3,4],
[6,5,7],
[4,1,8,3]
]
自顶向下的最小路径和为 11（即，2 + 3 + 5 + 1 = 11）。

说明：

如果你可以只使用 O(n) 的额外空间（n 为三角形的总行数）来解决这个问题，那么你的算法会很加分。

思路

动态规划，而且思路很经典，看完之后突然回想起之前的做LSTM+CRF的维特比算法，也是这个思想

相比于解决这个题，一个标准的动态规划的思路是更好的。之前都是上来就去定义状态，找状态转移方程，发现并没有那么好找。俗话说得好，万物皆套娃。大部分的dp问题都是由于DFS时间复杂度过高，进而转向带记忆优化的DFS，再进而优化为迭代的DP算法

蛮干

思路很简单，就是每次岔开左和右，取两者最小的值。

需要注意的点是，因为给定结构的特殊性，递归结束只需要判断 到达最后一层即可。由于每一次col都是原值或者+1到达下一层，不会越界的

带记忆的DFS

思路也很简单就是将没有算过的直接计算，然后存到mem里面，如果算过了就直接拿出来即可

其实从这个记忆化的DFS就可以感受到DP算法的韵味了。

这里面的每一次计算都有着一个递推公式，或者说是状态转移方程

$mem[level][col] = min(left,right)+triangle[level][col] = min(dfs(level+1,col),dfs(level+1,col+1)) + tri[level][col]$

其中的level 表示层数，col表示列数，简化为i,j

$mem[i][j] = min(mem[i+1][j],mem[i+1][j+1]) + triangle[i][j]$

上面的公式是由底向上的，完全根据dfs 过程推出来的

下面按照DFS过程写出动态规划的解法：

里面有一行去DFS过程极其相似，那就是$mem[i][j] = min(mem[i+1][j],mem[i+1][j+1]) + triangle[i][j]$

上面算法的时间复杂度和空间复杂度都是 o(N^2)的，但是我们发现里面的mem数组实际上利用效率很低，我们每次只使用上下两行的。其余都没有关系。

动态规划优化

这里使用了一个1维度的数组每次都是更新这一个数组就好了。

这是因为每次我们替换j位置的值，都是先根据之前该位置的值求得进行替换的，所以中间不会发生交叉。

思路

利用并查集的思想，题目很简单，相等的就是联通的，将他们合并在一起，不等的就不用合并。

最小的K个数

输入整数数组 arr ，找出其中最小的 k 个数。例如，输入4、5、1、6、2、7、3、8这8个数字，则最小的4个数字是1、2、3、4。

示例 1：

示例 2：

限制：

  • 0 <= k <= arr.length <= 10000
  • 0 <= arr[i] <= 10000

思路

经典面试题目：主要利用快排的partition 以及 堆排两种方式来完成

快排方案

原理： 快排一次可以确定一个数的最终位置，那么根据位置和k进行比较则可以在不完全排序好整个数组的前提下，排好前k个

方案分析：每次partition操作都是O(N)的时间复杂度，最优情况就是一次partition恰好找到第k个位置，则前K个就是topk小。最坏的情况就是每一次选出来的位置，不是最大就是最小，界限始终在两侧递减1，这样最终需要遍历完整个数组才能找到K，时间复杂度为O(N^2),平均的话 说不好，因为不是严格的快排序那种二分去找的。

堆排方案

维护一个大根堆，用来获取最小的前k个值，为什么用大根堆呢？

大根堆每次调整后，都将堆内最大的元素放到堆顶，如果新进来的元素比堆顶元素小，那么就把堆顶元素换成该元素，同时再进行一次从top到bottom的调整，这样最大的元素又一次被调整到堆顶，继续前面的过程。

最终堆内留下来的就是最小的K个数

时间复杂度分析：最坏情况就是o(nlogn) 对于数字需要进行完全的遍历o(N)，而每一次遍历都需要进行logN的操作。最优情况就是O(N)，也就是遍历过程不存在堆的调整，其实说白了就是最初加入堆的K个元素就是topK.
