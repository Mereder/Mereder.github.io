回文子串

具有不同开始位置或结束位置的子串，即使是由相同的字符组成，也会被计为是不同的子串。

示例 1:

输入: “abc”
输出: 3
解释: 三个回文子串: “a”, “b”, “c”.
示例 2:

输入: “aaa”
输出: 6
说明: 6个回文子串: “a”, “a”, “a”, “aa”, “aa”, “aaa”.

输入的字符串长度不会超过1000。

Leetcode-18-四数之和

Posted on 2019-12-08 In 算法刷题 , Leetcode

四数之和

给定一个包含 n 个整数的数组 nums 和一个目标值 target，判断 nums 中是否存在四个元素 a，b，c 和 d ，使得 a + b + c + d 的值与 target 相等？找出所有满足条件且不重复的四元组。

注意：

答案中不可以包含重复的四元组。

示例：

给定数组 nums = [1, 0, -1, 0, -2, 2]，和 target = 0。

满足要求的四元组集合为：
[
[-1, 0, 0, 1],
[-2, -1, 1, 2],
[-2, 0, 0, 2]
]

思路

跟三数之和思路是一样的，我们复习下：

三数之和中，我们先通过预排序，让数组有序。然后固定一个数，然后移动剩下的两个数（双指针）

即：预排序+固定一个+双指针

对于四数之和，我们采用同样的方案，只不过这里需要更改的是，我们固定两个数

  • 特判，特殊情况直接处理
  • 预排序
  • 固定一个数a
    • 处理相同情况
    • 再固定一个数b
      • 处理b的相同情况
      • 定义双指针
      • 双指针进行移动
        • 4sum == target添加结果
        • 4sum < target: 移动左指针
        • 4sum > target: 移动右指针

Posted on 2019-12-08 In 算法刷题 , Leetcode

两数之和

给定一个整数数组 nums 和一个目标值 target，请你在该数组中找出和为目标值的那 两个 整数，并返回他们的数组下标。

你可以假设每种输入只会对应一个答案。但是，你不能重复利用这个数组中同样的元素。

示例:

给定 nums = [2, 7, 11, 15], target = 9

因为 nums[0] + nums[1] = 2 + 7 = 9
所以返回 [0, 1]

思路

暴力解法，直接两层循环，输出相加为target的组合，时间复杂度O(N^2)

优化，采用空间换时间的策略，可以将数存储在字典(hash结构)中，然后直接索引就可以了

Leetcode-15-三数之和

Posted on 2019-12-08 In 算法刷题 , Leetcode

三数之和

给定一个包含 n 个整数的数组 nums，判断 nums 中是否存在三个元素 a，b，c ，使得 a + b + c = 0 ？找出所有满足条件且不重复的三元组。

注意：答案中不可以包含重复的三元组。

例如, 给定数组 nums = [-1, 0, 1, 2, -1, -4]，

满足要求的三元组集合为：
[
[-1, 0, 1],
[-1, -1, 2]
]

思路

暴力法，通过枚举所有的可能情况来看有没有可能和为0， 时间复杂度O(N^3)

本题解题思路体现了很重要的一点，当复杂度高于排序的时间复杂度时候，可以通过预排序来降低时间复杂度。

毕竟做一次排序的时间复杂度为O(NlogN)

所有数字排序之后，就有了很好的特性有序。

既然有序，我们想找三个数相加为0 ，首先确认一个数作为固定的，然后移动剩余两个数：

  • 3sum > 0 说明较大，则把较大一侧的数据向小方向挪一下
  • 3sum < 0 说明较小，则把较小一侧的数据向大方向挪一下
  • 3sum == 0 则找到了一组合适的

其中还有一些细节问题——不重复的数

由于排序之后，重复数字都是挨在一起的，我们以其中一个数作为固定数之后，其他的都可以直接跳过。

Posted on 2019-12-08 In 算法刷题 , Leetcode

K个不同整数的子数组

给定一个正整数数组 A，如果 A 的某个子数组中不同整数的个数恰好为 K，则称 A 的这个连续、不一定独立的子数组为好子数组。

（例如，[1,2,3,1,2] 中有 3 个不同的整数：1，2，以及 3。）

返回 A 中好子数组的数目。

示例 1：

输出：A = [1,2,1,2,3], K = 2
输入：7
解释：恰好由 2 个不同整数组成的子数组：[1,2], [2,1], [1,2], [2,3], [1,2,1], [2,1,2], [1,2,1,2].
示例 2：

输入：A = [1,2,1,3,4], K = 3
输出：3
解释：恰好由 3 个不同整数组成的子数组：[1,2,1,3], [2,1,3], [1,3,4].

提示：

1 <= A.length <= 20000
1 <= A[i] <= A.length
1 <= K <= A.length

思路

方案比较巧妙的办法是：

K个不同整数的子数组 = 用至多K个不同整数的结果 - 至多K-1个不同整数的结果

以【1，2，3】为例，K=3时候，

至多K个包含:[1],[2],[3],[1,2],[2,3],[1,3],[1,2,3]

至多K-1个包含: [1],[2],[3],[1,2],[2,3],[1,3]

做差结果 就是 [1,2,3]

Z字形变换

将一个给定字符串根据给定的行数，以从上往下、从左到右进行 Z 字形排列。

比如输入字符串为 “LEETCODEISHIRING” 行数为 3 时，排列如下：

L C I R
E T O E S I I G
E D H N
之后，你的输出需要从左往右逐行读取，产生出一个新的字符串，比如：”LCIRETOESIIGEDHN”。

请你实现这个将字符串进行指定行数变换的函数：

string convert(string s, int numRows);

示例 1:

输入: s = “LEETCODEISHIRING”, numRows = 3
输出: “LCIRETOESIIGEDHN”
示例 2:

输入: s = “LEETCODEISHIRING”, numRows = 4
输出: “LDREOEIIECIHNTSG”
解释:

E O E I I
E C I H N
T S G

思路

这种题上来就先模拟下这个的生成过程，模拟过程中就能找到一定的规律：

  • 先从上下下，再从下向上
  • 总是在 0行或者 n-1行转换反向
  • 不要受Z字形诱导，最后的输出结果就是按行从上到下将字符串拼接

举个例子：

LEETCODEISHIRING

结果是：

E D H N

最长回文子串

给定一个字符串 s，找到 s 中最长的回文子串。你可以假设 s 的最大长度为 1000。

示例 1：

输入: “babad”
输出: “bab”
注意: “aba” 也是一个有效答案。
示例 2：

输入: “cbbd”
输出: “bb”

思路

首先判断一个字符串是不是回文字符串，就是看该字符串是否是中心对称的。

  • 奇数个时: aba对称
  • 偶数个时：aa对称

现在是从一个字符串中，找出最长的回文字符子串，只需要遍历一遍字符串，假设每个位置都是一个回文子串的中心，然后根据是否是回文串来验证，记录长度最大的回文子串。

分析：总的遍历为o(N)，两种对称方式的遍历为o(2N)，总的时间复杂度为o(N^2)

给你一个字符串 S、一个字符串 T，请在字符串 S 里面找出：包含 T 所有字母的最小子串。

示例：

输入: S = “ADOBECODEBANC”, T = “ABC”
输出: “BANC”
说明：

滑动窗口适用解决的问题：
